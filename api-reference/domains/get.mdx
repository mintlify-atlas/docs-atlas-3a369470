---
title: "Get domain mapping"
description: "Retrieve a single domain mapping by domain name"
---

Returns the backend configuration for a specific domain.

## Authentication

All requests require an `Authorization: Bearer <PROXY_API_KEY>` header and must originate from the `PROXY_API_DOMAIN` host. See [Authentication](/api-reference/authentication) for details.

## Request

**GET** `/api/config/:domain`

<ParamField path="domain" type="string" required>
  The domain name to retrieve. URL-encode the value if it contains special characters.
</ParamField>

## Response

A single domain object, or `404` if the domain mapping does not exist.

<ResponseField name="domain" type="string" required>
  The domain name used to match incoming requests.
</ResponseField>

<ResponseField name="ip" type="string" required>
  The backend IP address traffic is forwarded to.
</ResponseField>

<ResponseField name="port" type="number" required>
  The backend port number.
</ResponseField>

<ResponseField name="protocol" type="string" required>
  The protocol used to reach the backend. Either `http` or `https`.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the mapping was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the most recent update to this mapping.
</ResponseField>

## Example

```bash
curl http://api.example.com/api/config/example.com \
  -H "Authorization: Bearer your-secret-api-key"
```

```json
{
  "domain": "example.com",
  "ip": "192.168.1.100",
  "port": 8080,
  "protocol": "http",
  "created_at": "2024-01-01T00:00:00Z",
  "updated_at": "2024-01-01T00:00:00Z"
}
```

## Error responses

| Status | Cause |
|--------|-------|
| `400 Bad Request` | The `domain` path parameter could not be URL-decoded. |
| `404 Not Found` | No mapping exists for the specified domain. |
