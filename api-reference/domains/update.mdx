---
title: "Update domain mapping"
description: "Update the backend configuration for an existing domain"
---

Changes the backend IP address, port, or protocol for a domain mapping that already exists.

## Authentication

All requests require an `Authorization: Bearer <PROXY_API_KEY>` header and must originate from the `PROXY_API_DOMAIN` host. See [Authentication](/api-reference/authentication) for details.

## Request

**PUT** `/api/config/:domain`

<ParamField path="domain" type="string" required>
  The domain name whose mapping you want to update. URL-encode the value if it contains special characters.
</ParamField>

<ParamField body="ip" type="string" required>
  The new backend IP address to forward traffic to.
</ParamField>

<ParamField body="port" type="number" required>
  The new backend port number.
</ParamField>

<ParamField body="protocol" type="string">
  The protocol used to reach the backend. Accepted values: `http`, `https`. If omitted, the existing protocol value is preserved.
</ParamField>

## Response

Returns `200 OK` with the updated domain object. The `updated_at` field is automatically set to the current timestamp.

<ResponseField name="domain" type="string" required>
  The domain name for this mapping.
</ResponseField>

<ResponseField name="ip" type="string" required>
  The updated backend IP address.
</ResponseField>

<ResponseField name="port" type="number" required>
  The updated backend port number.
</ResponseField>

<ResponseField name="protocol" type="string" required>
  The protocol used to reach the backend.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the mapping was originally created. Not changed by updates.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of this update.
</ResponseField>

## Example

```bash
curl -X PUT http://api.example.com/api/config/example.com \
  -H "Authorization: Bearer your-secret-api-key" \
  -H "Content-Type: application/json" \
  -d '{"ip":"192.168.1.200","port":9090,"protocol":"https"}'
```

```json
{
  "domain": "example.com",
  "ip": "192.168.1.200",
  "port": 9090,
  "protocol": "https",
  "created_at": "2024-01-01T00:00:00Z",
  "updated_at": "2024-01-15T12:00:00Z"
}
```

## Error responses

| Status | Cause |
|--------|-------|
| `400 Bad Request` | The `domain` path parameter could not be URL-decoded. |
| `400 Bad Request` | Request body is not valid JSON, or `ip` or `port` is missing. |
| `404 Not Found` | No mapping exists for the specified domain. |
