---
title: "Quickstart"
description: "Get Simple Proxy up and running in under 5 minutes"
---

This guide walks you through cloning the project, configuring the required environment variables, starting the proxy, and creating your first domain mapping.

<Steps>
  <Step title="Prerequisites">
    You need **Go 1.21 or later** installed on your machine. No CGO is required — Simple Proxy uses `modernc.org/sqlite`, a pure-Go SQLite driver that compiles without native libraries.

    Verify your Go version:

    ```bash
    go version
    ```
  </Step>

  <Step title="Clone and download dependencies">
    Clone the repository and fetch Go module dependencies:

    ```bash
    git clone https://github.com/ItsNoxius/simple-proxy
    cd simple-proxy
    go mod download
    ```
  </Step>

  <Step title="Set required environment variables">
    Simple Proxy requires two environment variables before it will start. The remaining variables are optional and have sensible defaults.

    ```bash
    export PROXY_API_KEY=my-secret-key
    export PROXY_API_DOMAIN=api.example.com
    export PORT=80
    export DB_PATH=data/proxy.db
    export DEBUG=false
    ```

    <Warning>
      `PROXY_API_KEY` is required. If it is not set, the server will exit immediately with a fatal error on startup. Choose a strong, randomly generated value for production use.
    </Warning>

    <Note>
      `PROXY_API_DOMAIN` controls which domain the management API is accessible from. The domain restriction middleware rejects any request to `/api/*` whose `Host` header does not match this value exactly. Set it to the domain you will use to reach the proxy's management interface — for example, `api.example.com`.
    </Note>
  </Step>

  <Step title="Run the proxy">
    Create the data directory for the SQLite database, then start the proxy:

    ```bash
    mkdir -p data
    go run main.go
    ```

    You should see output similar to:

    ```
    2024/01/01 00:00:00 [INFO] Starting server on port :80
    ```

    To build a standalone binary instead:

    ```bash
    go build -o proxy main.go
    ./proxy
    ```
  </Step>

  <Step title="Create a domain mapping">
    Use the REST API to map a domain name to a backend server. The request must be sent to the proxy using the domain you set in `PROXY_API_DOMAIN`, with your API key in the `Authorization` header.

    ```bash
    curl -X POST http://api.example.com/api/config \
      -H "Authorization: Bearer my-secret-key" \
      -H "Content-Type: application/json" \
      -d '{"domain":"example.com","ip":"192.168.1.100","port":8080,"protocol":"http"}'
    ```

    A successful response returns the created domain mapping:

    ```json
    {
      "domain": "example.com",
      "ip": "192.168.1.100",
      "port": 8080,
      "protocol": "http",
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z"
    }
    ```

    The `protocol` field is optional and defaults to `"http"` if omitted.
  </Step>

  <Step title="Test the proxy">
    Send a request to the proxy using the mapped domain in the `Host` header:

    ```bash
    curl -H "Host: example.com" http://localhost/
    ```

    Simple Proxy looks up `example.com` in its database and forwards the request to `http://192.168.1.100:8080/`. If no mapping exists for the requested domain, the proxy returns `404 Domain not found`.
  </Step>
</Steps>
