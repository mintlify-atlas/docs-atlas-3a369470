Simple Proxy is a Go-based HTTP reverse proxy that inspects the `Host` header of incoming requests and forwards them to the correct backend server. Domain-to-backend mappings are stored in SQLite and can be created, updated, or deleted at runtime through a REST API — no restarts required.

## How it works

Simple Proxy listens on a single port and routes every incoming HTTP request based on the `Host` header value:

1. A request arrives at the proxy
2. The proxy looks up the `Host` header value in its SQLite database
3. If a mapping exists, the request is forwarded to the configured `ip:port` using the specified protocol
4. If no mapping exists, the proxy returns a `404 Not Found` response

All domain mappings are managed through a protected REST API, so you can add or remove routes without restarting the proxy.

## Getting started

<Steps>
  <Step title="Set required environment variables">
    Export your API key and the domain from which the management API will be accessed:

    ```bash
    export PROXY_API_KEY=your-secret-api-key
    export PROXY_API_DOMAIN=api.example.com
    ```
  </Step>
  <Step title="Start the proxy">
    Run Simple Proxy using Go or Docker:

    ```bash
    go run main.go
    ```
  </Step>
  <Step title="Create your first domain mapping">
    Use the REST API to route a domain to a backend server:

    ```bash
    curl -X POST http://api.example.com/api/config \
      -H "Authorization: Bearer your-secret-api-key" \
      -H "Content-Type: application/json" \
      -d '{"domain":"example.com","ip":"192.168.1.100","port":8080,"protocol":"http"}'
    ```
  </Step>
  <Step title="Test the proxy">
    Send a request using the mapped domain:

    ```bash
    curl -H "Host: example.com" http://localhost/
    ```

    The proxy forwards this request to `http://192.168.1.100:8080/`.
  </Step>
</Steps>
